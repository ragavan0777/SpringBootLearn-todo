# 🛡️ GitHub vs. Local: Security Best Practices

When working on a professional Spring Boot project (or teaching students), it is critical to distinguish between **Public Code** and **Private Secrets**.

---

### 📤 What to Push to GitHub (Public)
These files should be shared with students and the world. They contain the *logic* of the application but no sensitive data.

| File / Folder | Why? |
| :--- | :--- |
| **`src/`** | Contains all Java logic, controllers, and services. |
| **`pom.xml`** | Defines dependencies and build settings. |
| **`Dockerfile`** | Instructions for building the container. |
| **`docs/`** | All documentation and learning guides. |
| **`application.properties`** | Shared configuration (uses `${VAR:default}` placeholders). |
| **`.gitignore`** | Tells Git which files to ignore (Crucial!). |

---

### 🏠 What to Keep Local (Private)
These files contain "Secrets" (passwords, API keys) or temporary computer-generated files. **NEVER** push these to GitHub.

| File / Folder | Why? |
| :--- | :--- |
| **`.env`** | Contains real database passwords and JWT secrets. |
| **`target/`** | Large, compiled Java files generated by Maven. GitHub only needs source code. |
| **`.idea/` or `.vscode/`** | Personal IDE settings that might conflict with others. |
| **`local_db.sql`** | Any data dumps containing real user information. |

---

### 🔑 The "Placeholder" Strategy
How do we share code without sharing passwords? We use **Environment Variables**.

In [application.properties](file:///home/loordhujeyakumar/Downloads/startSpring/src/main/resources/application.properties), we wrote:
```properties
spring.datasource.password=${DB_PASSWORD:MyLocalPassword}
```

- **GitHub sees**: "I need a password called `DB_PASSWORD`. If it's missing, use `MyLocalPassword`."
- **Render/Production**: You provide the *real* password in the Dashboard.
- **Security**: Your real cloud database password is never visible in the source code.

---

### 🎓 Teaching Point for Students
> "Treat your GitHub like a storefront. Everyone can see the layout and the products (the code), but only you should have the keys to the safe (the passwords)."
